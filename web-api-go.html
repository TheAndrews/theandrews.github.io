API to Manage Customers and Wishlists

API made with Go connecting to a PostgreSQL database.

This project utilizes concepts of DDD (Domain-Driven Design) and SOLID principles.

It utilizes uber-go/dig as a dependency injection container. From repositories to services and controllers, all components implement interfaces which are fulfilled by the container.

Key Features

  • Simple authentication that checks for an API key (header x-api-key)
  • Tests are present together with their implementation following Go test pattern
  • Uses mocks generated from interfaces (mockery package)
  • Repository-level tests are integration tests that hit the database using a Docker Postgres pod (testcontainers-go package)

Technical Details

Go version: 1.24

Packages utilized:

  • gin
  • gin-swagger
  • dig
  • gorm
  • gormigrate
  • mockery
  • testcontainers-go
